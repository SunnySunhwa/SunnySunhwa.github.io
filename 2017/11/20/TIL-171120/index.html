TIL_171120 Angular Service

@Injectable 데코레이터를 통해 순수한 자바스크립트 클래스가 다른 자바스크립트 클래스 내에 주입 가능한 상태가 되었다
Service는 Component와 Service를 사용할 수 있음
서비스를 사용한다는 이야기는 서비스 클래스가 생성한 인스턴스를 사용한다는 것
컴포넌트 내에 new를 사용하는 방법은 컴포넌트가 사용하는 서비스(의존성)가 구현하는 내용의 변경이 요구됐을때,
긴밀한 결합을 유지하고 있으므로, 다른 서비스로 교체를 해줘야 한다는 단점이 있음 -> 컴포넌트 내 변경해줘야 할 사항이 많음
또한, 인스턴스 생성방법을 컴포넌트가 알고 있어야한다는 단점이 있음

-> 해결방안
의존성 주입
컴포넌트가 사용하는 서비스(의존성)의 인스턴스를 컴포넌트에서 new로 생성하는 것이 아니라,
서비스에서 생성하여 그것을 컴포넌트에 주입하여 사용한다

providers에는 컨스트럭터에 매개변수로 선언한 서비스의 인스턴스가, 사용한 클래스 명을 적어줌.
즉, 해당 컴포넌트의 컨스트럭터 매개변수(인스턴스)의 타입은, providers에 적어준 클래스를 사용해서 만드는 것이라 알려주는 것
컨스트럭터 내에 private을 써주지 않으면, 멤버변수화가 되지 않아 컨스트럭터 내부에서만 사용할 수 있으므로 써주어야함 (타입스크립트)

인젝터는 컴포넌트별로 하나씩 가지고 있음
모듈에 프로바이더를 설정한 경우, 컴포넌트에 프로바이더를 또 설정하면 X -> 싱글턴이 유지되지 않음 (서비스가 두개 생성됨)
