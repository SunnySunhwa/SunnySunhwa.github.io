form - submit

  • form안의 요소들(그룹)이 하나라도 유효하지 않으면 전송할 수 없다.

form control

  • name 어트리뷰트가 데이터의 이름이 된다.
  • ngModel을 써줘야 control이 만들어짐
  • 각각의 input에 접근하기 위해서는 form 참조 변수를 사용한다.
  • ex) userForm.value (참조변수:#userForm)
  • #name=”ngModel”: 유효성 검증 상태 추적이 가능해진다.

템플릿 기반 폼 유효성 검증 - AbstractControl클래스

  • 유효성 검증의 상태를 나타내는 프로퍼티를 가지고 있음
  • errors, invalid, valid, pristine, undouched, dirty 등
  • untouched: fousin -> focusout되면 false
  • pristine: 값이 한번이라도 입력된 상태 false
  • *ngIf를 사용해서 에러메시지를 출력하는게 좋다 -> 조건에 부합하면 DOM에서 사라지기 때문
  • 에러메시지는 값을 입력하고 focusout됐을 때 출력되야 함.
  • errors => invalid: true, untouched: false

  • 컴포넌트 기반
  • formControlName이 템플릿기반 폼에서 name의 역할을 한다.

템플릿 기반 폼(Template-driven Forms)이란?

FormsModule을 적용하는 순간 모든 폼 요소는 템플릿 기반 폼이 된다. 그 기반을 쓰지 않으려면

이라고 써줘야 한다. 이외의 모든 폼은 템플릿 기반 폼을 따른다.

템플릿 기반 폼의 중심 디렉티브

  1. NgForm 디렉티브

    • (submit) 이벤트의 기본 동작을 방지해주기 위해 이벤트를 (ngSubmit)으로 사용한다.
    • 템플릿 참조 변수에는 ngForm을 할당한다. #은 선언의 의미를 갖는다. #f = “ngForm”
    • 유효성 검증을 하기 위해서 각각의 폼 컨트롤 요소에 ngModel 디렉티브를 선언해주어야 한다.
    • NgForm을 통해 받는 프로퍼티 controls에는 각 폼 요소의 name 어트리뷰트에 기반한 자료를 받는다. 따라서 name 어트리뷰트를 빼먹으면 안 된다.

  2. NgModel 디렉티브

  • 폼 컨트롤 요소를 위해 사용
  • 적용된 폼 컨트롤 요소에 FormControl 인스턴스 생성

NgModel과 양방향 바인딩

  • 템플릿 기반 폼 유효성 검증

  • NgForm, NgModel, NgModelGroup이란 것은 해당 디렉티브가 선언된 요소에 대한 객체를 만들어주는 것.
  • 유효성 검증 상태 프로퍼티
    • pristine: true 라면 한번도 입력되지 않았기 때문에 에러를 띄우면 안 된다.
